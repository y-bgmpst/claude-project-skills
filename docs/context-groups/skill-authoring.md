# Skill Authoring

Use this group when adding or editing a skill.

## Keep skills small

- one skill = one narrow job
- prefer direct instructions over long exposition
- move repeated or detailed guidance into a supporting file only if needed

## Claude skill shape

- `Skill.md` is the source of truth
- keep the description specific enough to trigger correctly
- keep the instructions short and action-oriented

## Good edits

- tighten the trigger description
- reduce overlap with another skill
- add one concise workflow
- remove stale or redundant guidance

## Bad edits

- merging unrelated tasks into one skill
- bloating the skill body with wiki-level detail
- adding hidden dependencies that are not documented
