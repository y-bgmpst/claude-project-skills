# Release

Use this group when preparing a release, tag, or draft release text.

## Release contents

- what changed
- which skills were added or modified
- whether install or validation changed
- whether the wiki or routing changed

## Release style

- keep the first release small
- write a short summary plus a compact bullet list
- include validation commands only if they matter to the change

## Publishing checklist

- confirm `main` is green
- confirm install still works
- confirm the release text matches the repository contents
