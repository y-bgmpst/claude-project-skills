# Claude Bootstrap

Start here when opening a new repo with Claude.

1. Read `project.md`.
2. Read `docs/context-groups/index.md`.
3. Load only the smallest matching context group.
4. Keep prompts short and move reusable detail into repo files.

## Default routing

- repo setup, install, validation -> `docs/context-groups/repo-setup.md`
- skill edits -> `docs/context-groups/skill-authoring.md`
- safety / mutations -> `docs/context-groups/safety.md`
- releases -> `docs/context-groups/release.md`
- Kubernetes / Docker / container ops -> `docs/context-groups/container-ops.md`
- common user tasks -> `docs/context-groups/common-tasks.md`

## Mac rule

On macOS, prefer the editor or agent workflow that already exists in the repo; do not invent extra tools unless they are needed for the task.
